This library is based on work on the incomplete jsgui-client-webkit-ios. Within that library lots of development on data types and nested get/set, and that is being spun off into its own file. It can be tested separately better that way, and the codebase could become a lot clearer by removing it from that file and using a modular pattern. It would make a useful library and will be released as open-source for that purpose. Others may wish to develop it further. It will be a lot like backbone.js and will take cues from it in some ways. May have some or all of the backbone.js API.

Controls will use the language features. Will also have the HTML composition. The HTML module will be the basis for using HTML controls.

The jsgui object - this will be created by lang, and extended by other libraries. Not allowed globals? We want one global though, maybe there ill be a way around it.
